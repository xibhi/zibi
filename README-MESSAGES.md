# Zibi Message Management System

A comprehensive message management system for the zibi clipboard manager that automatically cycles through success and error messages for each command.

## Overview

This system reads from `success.txt` and `error.txt`, manages message rotation per command, and displays contextually appropriate messages when commands complete or encounter errors.

### Features

- **Message Cycling**: Each command rotates through its available messages, ensuring variety in feedback
- **Placeholder Replacement**: Dynamic replacement of variables like `{n}`, `{max}`, `{path}`, `{mode}`, etc.
- **State Tracking**: Automatically tracks which message to display next via `.message-state.json`
- **Error Type Support**: Different error scenarios for the same command use different message sets
- **Alternative Command Names**: Commands with "/" (like `--fetch / recall`) have aliased message access

## Files

### Main Scripts

- **`zibi-messages.ps1`** - Core message engine
  - Parses success/error files
  - Manages message state
  - Performs placeholder replacements
  
- **`zibi-command.ps1`** - User-friendly wrapper
  - Simplifies calling messages
  - Handles data formatting
  - Displays colored output

### Configuration Files

- **`success.txt`** - Success messages for all commands
- **`error.txt`** - Error messages for all error types
- **`.message-state.json`** - Auto-generated state file (do not edit manually)

## Usage

### Basic Success Message

```powershell
& '.\zibi-command.ps1' -Command '--copy' -OnSuccess 1
```

Output:
```
Snatched. zibi is holding it tight.
```

Next call will show:
```
Locked in. zibi won't let go.
```

### Error Message

```powershell
& '.\zibi-command.ps1' -Command '--fetch' -OnSuccess 0 -ErrorType '/ recall index out of range' -Data @{n=5; max=3}
```

Output:
```
Index 5 doesn't exist. zibi only has 3 entries.
```

### Error with Placeholder Replacement

```powershell
& '.\zibi-command.ps1' -Command '--copy' -OnSuccess 0 -ErrorType '--file file not found' -Data @{path='config.ini'}
```

Output:
```
zibi looked for 'config.ini' and found absolutely nothing.
```

## State Management

The `.message-state.json` file automatically tracks the next message index for each command:

```json
{
  "success:--copy": 1,
  "success:--clear": 0,
  "error:--fetch / recall index out of range": 2,
  "error:--copy --file file not found": 1
}
```

**Important**: Do not edit this file manually. It is auto-generated and manages the cycling state.

## Message Placeholders

Supported placeholders in messages:

| Placeholder | Used In | Example |
|-------------|---------|---------|
| `{n}` | Index out of range errors | "Index {n} doesn't exist" |
| `{max}` | Range errors | "zibi only has {max} entries" |
| `{path}` | File not found errors | "zibi looked for '{path}'" |
| `{mode}` | Transform unknown mode | "'{mode}' is not a real mode" |
| `{val}` | Invalid index type | "zibi needs a number, not '{val}'" |
| `{query}` | Search no results | "Nothing found for '{query}'" |

## Integration

To integrate with actual zibi commands, wrap your command execution like this:

```powershell
# Try the command
try {
    # Your zibi command here
    $result = & zibi --copy "some text"
    
    # On success, show a success message
    & '.\zibi-command.ps1' -Command '--copy' -OnSuccess 1
    Write-Host "Result: $result"
}
catch {
    # On error, show an appropriate error message
    & '.\zibi-command.ps1' -Command '--copy' -OnSuccess 0 -ErrorType 'no input given'
}
```

## Troubleshooting

### Message not found error

Ensure the error type matches exactly. For commands with "/" like `--fetch / recall`, use:
```powershell
-ErrorType '/ recall index out of range'
```

Not:
```powershell
-ErrorType 'index out of range'  # This won't work
```

### Placeholders not replacing

Make sure your data hashtable keys match the placeholder names (without braces):

```powershell
# Correct
-Data @{n=5; max=3}

# Incorrect
-Data @{'{n}'=5; '{max}'=3}
```

### State not cycling

Delete `.message-state.json` to reset all message states:

```powershell
Remove-Item '.\message-state.json'
```

## Message File Format

Messages are organized in plain text with this format:

```
# Command Name or Error Type
"First message variant"
"Second message variant"
"Third message variant"

# Another Command
"Message here"
```

For commands with alternatives, use "/":
```
# --fetch / recall index out of range
"Message 1"
"Message 2"
```

This creates aliases so both `--fetch index out of range` and `recall index out of range` access the same messages.
