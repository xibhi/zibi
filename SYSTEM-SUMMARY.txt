# Zibi Message Management System - Summary

## What Was Created

A complete message management system for the zibi clipboard manager that handles success and error messages with automatic cycling through multiple variants.

### Core Components

1. **zibi-messages.ps1** - Core engine that:
   - Parses success.txt and error.txt files
   - Manages message state persistence
   - Performs placeholder replacements
   - Handles command aliases (e.g., `--fetch / recall`)

2. **zibi-command.ps1** - User-friendly wrapper that:
   - Provides simple interface for displaying messages
   - Handles colored output for errors
   - Manages data/placeholder injection
   - Supports both success and error modes

3. **Documentation**:
   - README-MESSAGES.md - Comprehensive guide
   - QUICK-REFERENCE.txt - Quick lookup guide
   - test-messages.ps1 - Full test suite

## Key Features

### ✓ Message Cycling
Each command automatically cycles through its available messages:
```
Call 1: "Snatched. zibi is holding it tight."
Call 2: "Locked in. zibi won't let go."
Call 3: "Yours. Well, zibi's. Same thing."
Call 4: (cycles back to Call 1)
```

### ✓ Placeholder Replacement
Dynamic variable substitution:
```powershell
-Data @{path='config.ini'} 
# "zibi looked for 'config.ini' and found absolutely nothing."

-Data @{n=5; max=3}
# "Index 5 doesn't exist. zibi only has 3 entries."
```

### ✓ State Tracking
Automatic persistence via `.message-state.json`:
```json
{
  "success:--copy": 1,
  "error:--fetch / recall index out of range": 2,
  "error:--transform unknown mode": 0
}
```

### ✓ Error Type Support
Different messages for different error scenarios:
```powershell
--copy no input given
--copy --file file not found
--fetch / recall index out of range
--transform unknown mode
...and more
```

## Usage Examples

### Success Messages
```powershell
# Display a success message for --copy (cycles through variants)
& '.\zibi-command.ps1' -Command '--copy' -OnSuccess 1
```

### Error Messages
```powershell
# Display error with no placeholders
& '.\zibi-command.ps1' -Command '--copy' -OnSuccess 0 -ErrorType 'no input given'

# Display error with placeholder replacement
& '.\zibi-command.ps1' -Command '--fetch' -OnSuccess 0 -ErrorType '/ recall index out of range' -Data @{n=5; max=3}

# File not found with path
& '.\zibi-command.ps1' -Command '--copy' -OnSuccess 0 -ErrorType '--file file not found' -Data @{path='test.txt'}

# Transform with unknown mode
& '.\zibi-command.ps1' -Command '--transform' -OnSuccess 0 -ErrorType 'unknown mode' -Data @{mode='backwards'}
```

## Integration

To use with actual zibi commands:

```powershell
# Execute zibi command
try {
    $result = & zibi --copy "some text"
    
    # Show success message
    & '.\zibi-command.ps1' -Command '--copy' -OnSuccess 1
}
catch {
    # Show error message
    & '.\zibi-command.ps1' -Command '--copy' -OnSuccess 0 -ErrorType 'no input given'
}
```

## Reset Instructions

To clear all message cycling states and start fresh:
```powershell
Remove-Item 'x:\Downloads\zibi\.message-state.json'
```

The state file will be automatically recreated on next use.
