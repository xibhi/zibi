# Shell Integration & Installation Guide

The `zibi --init` command provides automatic shell integration, making zibi accessible from anywhere in your system.

## Quick Start

### Automatic Installation (Recommended)

```bash
zibi --init
```

This command will:
1. **Auto-detect your shell** (bash, zsh, or fish)
2. **Install hooks** to your shell's config file
3. **Make zibi globally accessible** from any directory

That's it! Restart your shell or open a new terminal.

## What Gets Installed

The `--init` command adds shell hooks that:

1. **Enable `zibi_run` wrapper** - Captures command output
   ```bash
   zibi_run pytest -q        # Runs pytest and captures output
   zibi --snatch             # Retrieves the captured output
   ```

2. **Enable `run` alias** - Short alias for `zibi_run`
   ```bash
   run npm test              # Same as zibi_run npm test
   ```

3. **Track last command output** - Stores in `~/.cache/zibi/last_output.txt`

4. **Set up environment variables** - `ZIBI_LAST_OUTPUT` path

## Installation by Shell

### Bash

```bash
# Automatic (writes to ~/.bashrc)
zibi --init --shell bash

# Manual (just see the script)
zibi --init --shell bash --no-persist
zibi --init --shell bash --no-persist >> ~/.bashrc
```

Then reload:
```bash
source ~/.bashrc
```

### Zsh

```bash
# Automatic (writes to ~/.zshrc)
zibi --init --shell zsh

# Manual
zibi --init --shell zsh --no-persist >> ~/.zshrc
source ~/.zshrc
```

### Fish

```bash
# Automatic (writes to ~/.config/fish/conf.d/zibi.fish)
zibi --init --shell fish

# Manual
mkdir -p ~/.config/fish/conf.d
zibi --init --shell fish --no-persist > ~/.config/fish/conf.d/zibi.fish
```

## Command Options

### --shell
Specify which shell to install for:
- `auto` (default) - Auto-detect from `$SHELL` or parent process
- `bash` - Bash shell
- `zsh` - Zsh shell
- `fish` - Fish shell

```bash
zibi --init --shell bash
zibi --init --shell zsh
zibi --init --shell fish
```

### --persist / --no-persist
Control whether to automatically write to config file:
- `--persist` (default) - Write to shell config file (~/.bashrc, ~/.zshrc, etc.)
- `--no-persist` - Just print the script (for manual installation)

```bash
# Automatic (default)
zibi --init

# Manual (just print, no file write)
zibi --init --no-persist
```

## What Gets Added to Config File

A section like this is added to your shell's config file:

```bash
# === ZIBI START ===
# zibi shell integration for bash and zsh
# Generated by: zibi --init
# ... (shell integration code)
# === ZIBI END ===
```

The markers (`ZIBI START` and `ZIBI END`) allow zibi to update the installation if you run `--init` again.

## Usage Examples

### Capture and Snatch Command Output

```bash
# Run a command with zibi capturing the output
zibi_run pytest tests/
zibi_run npm test

# Later, retrieve the output
zibi --snatch

# Or see last copied item
zibi --fetch 1
```

### Using the `run` Alias

```bash
run python script.py
run cargo build --release
```

### Check Installation

After installation, verify it worked:

```bash
# This should show the zibi functions and aliases
which zibi_run

# This should show the cache directory
echo $ZIBI_LAST_OUTPUT
```

## Accessing zibi from Anywhere

After `zibi --init`, you can use zibi from **any directory** in your system:

```bash
# From your home directory
cd ~
zibi --count

# From a project directory
cd ~/projects/my-app
zibi --log

# From a deeply nested directory
cd ~/projects/my-app/src/components/utils
zibi --fetch 1

# From /tmp or anywhere else
cd /tmp
zibi --spy
```

## Troubleshooting

### Command Not Found

If you get "command not found: zibi", try:

```bash
# Reload your shell config
source ~/.bashrc    # for bash
source ~/.zshrc     # for zsh
exec fish          # for fish

# Or open a new terminal window
```

### Not Installing Properly

Check if the config file is writable:

```bash
# For bash
ls -la ~/.bashrc
chmod 644 ~/.bashrc

# For zsh
ls -la ~/.zshrc
chmod 644 ~/.zshrc

# For fish
ls -la ~/.config/fish/conf.d/
chmod 755 ~/.config/fish/conf.d/
```

### Python Path Issues

If zibi can't be found from your shell, the Python environment might not be in PATH. Verify:

```bash
which python3
which pip

# Check if zibi is installed
pip list | grep zibi
```

If missing, reinstall:
```bash
pip install -e .  # or pip install zibi
```

### Check Installation Details

```bash
# See which shell was detected
echo $SHELL

# See the zibi installation
cat ~/.bashrc | grep -A 50 "ZIBI START"

# Or for fish
cat ~/.config/fish/conf.d/zibi.fish
```

## Manual Installation

If `--persist` doesn't work for you, install manually:

```bash
# For bash
zibi --init --shell bash --no-persist >> ~/.bashrc
source ~/.bashrc

# For zsh
zibi --init --shell zsh --no-persist >> ~/.zshrc
source ~/.zshrc

# For fish
mkdir -p ~/.config/fish/conf.d
zibi --init --shell fish --no-persist > ~/.config/fish/conf.d/zibi.fish
```

## Re-running --init

You can run `zibi --init` multiple times safely:

```bash
zibi --init        # First time - installs
# ... later ...
zibi --init        # Second time - updates existing installation
```

The markers (`ZIBI START` and `ZIBI END`) ensure that only one copy of the hooks is kept, even if you run the command multiple times.

## Uninstalling

To remove zibi from your shell:

1. Open your config file in an editor
   ```bash
   nano ~/.bashrc      # bash
   nano ~/.zshrc       # zsh
   nano ~/.config/fish/conf.d/zibi.fish  # fish
   ```

2. Find the section marked `# === ZIBI START ===` to `# === ZIBI END ===`

3. Delete those lines

4. Save and reload:
   ```bash
   source ~/.bashrc    # bash
   source ~/.zshrc     # zsh
   exec fish          # fish
   ```

## Advanced Features

### Using with Multiple Shells

You can install zibi for multiple shells:

```bash
zibi --init --shell bash
zibi --init --shell zsh
zibi --init --shell fish
```

Each will use its own config file and work independently.

### Environment Variables

After installation, these are available:

```bash
$ZIBI_LAST_OUTPUT      # Path to the last captured output
$ZIBI_CACHE_DIR        # zibi's cache directory (~/.cache/zibi)
```

### Custom Aliases

The installation includes these functions:

```bash
zibi_run      # Main capture wrapper
run           # Alias for zibi_run
zibi_alias_common  # Setup common command aliases (pytest, npm, cargo, etc.)
```

To use common aliases:
```bash
zibi_alias_common
pytest tests/    # Now captured by zibi
```
