
## How to Use

No additional configuration needed! The system works automatically when you run zibi commands.

### Test It

```bash
# Run the same command multiple times to see messages cycle
zibi --copy "hello"
zibi --copy "world"
zibi --copy "test"

# Messages will cycle through the available variants
```

### Reset Message State

To start from the first message again for all commands:

```powershell
Remove-Item 'x:\Downloads\zibi\.zibi-messages\.message-state.json'
```

## System Architecture

```
zibi command execution
    ↓
main.py: --copy handler
    ↓
print_success(msg, command="--copy")
    ↓
utils.py: print_success()
    ↓
message_manager.py: get_success_message("--copy")
    ↓
Parse: success.txt → [msg1, msg2, msg3]
Load: .message-state.json → index: 1
Return: msg2 "Locked in. zibi won't let go."
Update: .message-state.json → index: 2
    ↓
Display with green panel
```

## Message Format

Messages are organized in simple text files:

```
# Command Name
"First variant"
"Second variant"
"Third variant"

# Another Command
"Single message"
```

Commands with alternatives use "/" notation:
```
# --fetch / recall index out of range
"Message 1"
"Message 2"
```

This creates aliases so multiple command names access the same messages.

## Placeholders Supported

Messages can include placeholders that get replaced dynamically:

- `{n}` - Index number
- `{max}` - Maximum count
- `{path}` - File path
- `{mode}` - Transform mode
- `{query}` - Search query
- `{val}` - Invalid value

Example:
```
# --fetch / recall index out of range
"Index {n} doesn't exist. zibi only has {max} entries."

Output: "Index 5 doesn't exist. zibi only has 3 entries."
```

## State Management

Message state is automatically persisted in:
```
x:\Downloads\zibi\.zibi-messages\.message-state.json
```

This file tracks which message variant to display next for each command.

Example state file:
```json
{
  "success:--copy": 1,
  "success:--pin": 0,
  "success:--clear": 1,
  "success:--transform": 0
}
```

**Important**: Do not edit this file manually. It's auto-generated and managed by the system.

## Troubleshooting

### Messages not showing

1. **Check installation**: Reinstall zibi
   ```bash
   cd x:\Downloads\zibi
   pip install -e . --force-reinstall --no-deps
   ```

2. **Check message files**: Verify files exist
   ```
   x:\Downloads\zibi\.zibi-messages\success.txt
   x:\Downloads\zibi\.zibi-messages\error.txt
   ```

3. **Check Python path**: Ensure message_manager.py is in zibi/
   ```
   x:\Downloads\zibi\zibi\message_manager.py
   ```

### State file corrupted

Delete and regenerate:
```powershell
Remove-Item 'x:\Downloads\zibi\.zibi-messages\.message-state.json'
```

The system will create a new one on next command execution.
